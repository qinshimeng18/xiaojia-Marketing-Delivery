# 三平台内容营销方法论与合规表达调研

**调研日期：2026-05-25**
**用途：给“小加同学”作为内容营销生成、改写、审核前置检查的操作手册。**
**重要说明：** 平台算法细节不会完整公开，本文的分发逻辑部分基于平台公开规则、公开创作者资料、行业公开案例与内容运营实践整理；合规部分优先参考平台规则与《广告法》《互联网广告管理办法》等公开法规。本文不是法律意见。遇到医疗、金融、教育、医美、药品、保健食品、未成年人等高敏行业，生成前应要求业务方提供资质、证据材料和最终法务/合规确认。

**对“小加同学”的总原则：** 不要用谐音、拆字、错别字、特殊符号去“绕审核”。正确做法是把文案从语义上改成真实、可证明、有限定条件、尊重用户选择的表达。

## 1. 总览

### 1.1 三个平台的核心差异

| 平台 | 核心流量逻辑 | 适合内容 | 内容气质 | 最容易踩的坑 |
| --- | --- | --- | --- | --- |
| 小红书 | 搜索 + 推荐 + 收藏沉淀 | 生活方式种草、测评、清单、避坑、教程、本地探店、AI/SaaS 轻体验 | 像朋友经验：具体、真实、有边界 | 夸大功效、虚假体验、站外引流、诱导互动、价格误导 |
| 微信 | 关注关系 + 私域转化 + 社群/朋友圈扩散 | 品牌故事、深度科普、产品说明、活动预告、客户案例、复购唤醒 | 像可信顾问：解释清楚、证据充分、CTA 克制 | 诱导分享/关注、虚假宣传、强承诺、客服/模板消息骚扰、洗稿侵权 |
| 抖音 | 短视频兴趣推荐 + 完播/停留/互动 + 搜索/同城 | 口播、探店、测评、教程、剧情、前后对比、直播切片、热点借势 | 像现场演示：3秒抓住、过程可见、结果克制 | 极限词、虚假宣传、功效承诺、诱导互动、低俗擦边、搬运侵权 |

### 1.2 每个平台适合什么内容

- **小红书：** 适合“用户决策前搜索”的内容，尤其是种草、避坑、测评、清单、教程、体验分享、对比笔记。AI 写作时要像“有真实体验的朋友”，优先输出场景、细节、边界和可执行清单。
- **微信：** 适合“建立信任后转化”的内容，包括品牌故事、深度科普、产品说明、活动预告、客户案例、私域转化、老客复购。AI 写作时要像“可信顾问”，重视证据、结构、案例、CTA 的克制。
- **抖音：** 适合“被动刷到也能理解”的内容，包括口播、探店、测评、教程、剧情、前后对比、清单、直播切片、热点借势。AI 写作时要像“现场演示”，优先设计前 3 秒、镜头、字幕、过程证据和评论问题。

### 1.3 内容创作时最容易踩的坑

1. 把体验写成保证：如“保证瘦”“一定有效”“包过”，应改成“个人体验/适用场景/结果因人而异”。
2. 把主观评价写成绝对排名：如“全网第一”“最佳”“最强”，应改成“在某场景下更适合”。
3. 把普通产品写成医疗、药品、医美效果：如“治失眠”“根治痘痘”，应改成“护理经验/健康管理建议”，并提示咨询专业人士。
4. 把用户互动做成任务：如“评论 666 发资料”“转发领取”，应改成自然提问或官方入口。
5. 把转化写成强导流：如“私信拿链接”“加微信下单”，应优先使用平台内官方入口、公开说明或合规客服入口。
6. 把价格写成无法证明的极值：如“最低价”“最后一天”，应写清活动时间、适用条件、库存和以官方公示为准。
7. 把案例写成普遍承诺：客户案例必须标注单个案例、统计口径、适用边界。
8. 搬运、洗稿、盗图、未授权音乐：一律改为原创/授权素材，并注明来源。

## 2. 小红书内容方法论

### 2.1 平台特点

小红书内容更接近“搜索型种草 + 兴趣推荐”的混合场景。用户常在购买、出行、变美、装修、选工具前主动搜索关键词，也会在发现页被相似兴趣内容推荐。公开运营资料普遍强调标题、正文、话题、评论中的关键词布局会影响搜索收录与精准推荐；平台社区规范则明确约束虚假宣传、价格欺诈、侵犯知识产权、高风险类内容等行为。

**AI 执行规则：**
- 先判断该笔记是“搜索型”还是“推荐型”。搜索型优先关键词；推荐型优先情绪、场景和冲突。
- 每篇笔记至少包含 1 个核心关键词、2 个场景关键词、1 个行业或人群关键词。
- 避免泛泛种草；必须写出“我为什么需要它、我怎么用、适合谁、不适合谁”。

### 2.2 爆款内容结构

**通用公式：**
`人群 + 场景 + 痛点 + 结果 + 情绪价值 + 适用边界`

**可执行结构：**
```text
标题：{人群/场景} + {痛点} + {有限定结果}
封面：6-10 字结论 + 场景图/对比图
开头：身份/场景 → 痛点 → 结论
正文：3-5 个模块，每个模块=观察/证据/动作
结尾：适用边界 + 自然提问
标签：品类词 + 场景词 + 人群词 + 地点/行业词
```

**小红书内容创作规则（至少 10 条）：**

1. 选题必须写成「人群 + 场景 + 痛点」，禁止只写品类词。例如不要写“咖啡店推荐”，改写成“周末带爸妈在静安吃饭，怎么避开排队和踩雷”。
2. 标题前 10 个字放核心关键词或场景词；如果是搜索型内容，标题中必须出现用户会搜的词，如“敏感肌粉底”“大理三天两晚”“AI 周报工具”。
3. 每篇只解决一个问题；正文最多拆 3–5 个要点，避免把测评、教程、活动、品牌故事塞进同一篇。
4. 开头三句固定为：我是谁/什么场景 → 遇到什么问题 → 本篇给什么结论；不使用“姐妹们冲”“不看后悔”等强刺激词。
5. 所有推荐必须写适用边界：适合谁、不适合谁、预算/时间/地域/肤质/基础条件是什么。
6. 测评笔记必须列评价维度，至少 3 项：价格、体验、稳定性、服务、售后、易用性等；不要只写“好用”。
7. 避坑笔记先写坑点，再写替代动作；每个坑点都要对应“怎么做”，不能只有吐槽。
8. 本地生活/餐饮笔记必须包含：位置或商圈、到店时间、排队情况、人均区间、适合人群、是否适合拍照或带长辈/孩子。
9. AI 工具/SaaS 笔记必须包含：使用场景、上手成本、一个具体工作流、限制条件、是否适合团队/个人。
10. 话题标签只放 3–6 个，按「品类词 + 场景词 + 人群词 + 地点词」组合；不堆无关热门标签。
11. 结尾只能做自然提问或信息补充，不写“点赞收藏关注三连”“私信发链接”。
12. 发布前执行合规扫描：绝对化、功效承诺、医疗/金融/教育敏感、价格承诺、站外导流、诱导互动各扫一遍。

### 2.3 标题写法

**标题公式：**
1. `{人群} + {场景} + {痛点} + {结果}`
   示例：新手装修前，先避开这 5 个收纳坑
2. `{预算/时间/地点} + {清单} + {适合人群}`
   示例：人均 100 内请客，这类店更稳
3. `{对象A} vs {对象B} + {选择标准}`
   示例：Notion vs 飞书，我会按这 4 个流程选
4. `{误区}不是{表面原因}，而是{真实原因}`
   示例：小户型越住越乱，不是柜子少

**不要写：** 全网最全、必买、闭眼入、永远不踩雷、保证有效、私信发链接。
**改成：** 我整理的版本、适合某类人、我会优先看、这几个点供参考。

### 2.4 封面写法

**封面规则：**
- 字数控制在 6–12 字，最多两行。
- 封面只表达一个结论，如“先看动线”“请客更稳”“3天不累路线”。
- 封面图优先使用真实场景、前后对比、清单截图、手写标注、实拍过程。
- 不使用“震惊”“不看后悔”“全网第一”等强刺激词。
- 美业、医美、健康类不放夸张前后对比，不暗示确定疗效。

### 2.5 正文写法

**正文模块模板：**
```text
开头：我在{场景}遇到{痛点}，这次按{维度}试了一遍。
模块1：先说结论：{结论}。证据：{体验/数据/观察}。
模块2：适合谁：{人群/预算/基础条件}。
模块3：不适合谁：{边界/限制}。
模块4：如果你要试，先确认{检查项1}、{检查项2}、{检查项3}。
结尾：以上是我的体验，不代表所有人都一样。你更在意{A}还是{B}？
```

**正文检查清单：**
- 是否有真实场景？
- 是否有适用边界？
- 是否有价格/活动条件说明？
- 是否删除“保证”“最”“第一”“根治”“包过”等强风险词？
- 是否避免私信导流和诱导互动？

### 2.6 话题标签策略

**标签组合：**
- 1 个品类标签：#咖啡店 #小户型装修 #AI工具
- 1–2 个场景标签：#周末去哪儿 #通勤穿搭 #论文写作
- 1 个人群标签：#敏感肌 #新手装修 #职场新人
- 1 个地点或行业标签：#上海探店 #SaaS选型 #大理旅行

**AI 禁止动作：**
- 不堆超过 8 个标签。
- 不蹭无关热点标签。
- 不把标签当引流口令，如 #私信领取 #加我拿资料。

### 2.8 敏感词与风险表达

小红书最常见风险是“种草体验被写成确定功效”“商业合作未通过合规链路”“站外导流”“诱导点赞收藏评论”。平台社区规范对虚假宣传、价格欺诈、知识产权、高风险行业内容均有约束；蒲公英合作内容也要求先审后发并遵守社区公约、行业与内容规范。

| 风险类型 | 风险表达 | 风险原因 | 推荐替代表达 | 示例句子 |
| --- | --- | --- | --- | --- |
| 绝对化用语 | 全网第一 | 缺少可核验证据，可能构成绝对化或虚假宣传 | 我用下来比较顺手 / 在同类体验里表现突出 | 这款工具在我这类轻量剪辑场景里比较顺手，适合先试用。 |
| 功效承诺 | 最有效 | 把主观体验包装成确定性结论，容易误导消费者 | 对我这个场景更有帮助 / 个人体验更明显 | 连续使用一周后，我在通勤收纳这个场景里感觉更省时间。 |
| 健康减重 | 保证瘦 | 对身体效果做保证，且可能涉及健康/医疗风险 | 体重管理思路 / 结果因饮食运动和个人情况不同 | 这是我记录饮食和运动后的体重管理过程，结果因人而异。 |
| 美妆功效 | 7天变白 | 美白功效需有依据，短期结果承诺风险高 | 肤色观感更均匀 / 妆感更干净 | 我个人使用后觉得妆面更干净，具体肤感因肤质不同。 |
| 医疗功效 | 根治痘痘 | 普通产品不得宣传疾病治疗或暗示医疗功效 | 改善痘痘困扰的护理思路 / 痘肌护理经验 | 这篇只分享我的痘肌护理经验，不替代医生建议。 |
| 权威背书 | 医生推荐必做 | 无资质或未授权背书易构成误导；医疗身份更敏感 | 建议咨询专业人士 / 以正规机构建议为准 | 如果涉及皮肤问题，建议先咨询专业人士再决定。 |
| 安全断言 | 亲测无副作用 | 个体体验不能代表安全性保证 | 我个人未出现明显不适 / 不适请停止并咨询 | 我个人使用时未出现明显不适，敏感肌建议先做局部测试。 |
| 强刺激营销 | 不买后悔一辈子 | 制造焦虑、强迫感，容易被判定为夸张诱导 | 适合这类人重点看看 / 可以作为备选 | 预算有限但需要通勤包的人，可以把它放进备选清单。 |
| 价格误导 | 价格最低 | 价格随时间平台变化，无法证明时易误导 | 近期看到的价格区间 / 以门店或官方页面为准 | 我看到的活动价在这个区间，具体以门店当天公示为准。 |
| 促销误导 | 最后一天 | 虚假限时会误导购买决策 | 活动时间以官方公告为准 / 本期活动到指定日期 | 本期活动到 6 月 1 日，权益以官方页面显示为准。 |
| 引流风险 | 私信我发链接 | 平台可能限制站外导流、私下交易或非官方引流 | 可在官方渠道查看 / 评论区交流使用问题 | 我把关键参数写在图 3，有问题可以评论区交流。 |

| 避坑绝对化 | 100%避坑 | 无法保证所有人都不踩坑 | 我踩过的几个坑，供参考 | 这是我第一次装修踩过的 5 个坑，供同户型参考。 |
| 标题党 | 绝对不踩雷 | 绝对化承诺，且容易形成标题党 | 我认为更稳妥 / 适合新手先看 | 这几项是我认为新手更稳妥的选择标准。 |
| 唯一性宣传 | 小红书唯一推荐 | 无法证明平台唯一，易造成误导 | 本篇只分享我实际体验过的一种方案 | 本篇只分享我实际体验过的一种方案，欢迎补充其他选择。 |
| 非医疗品医疗化 | 普通枕头治失眠 | 非医疗产品不得暗示治疗疾病 | 改善睡眠环境 / 睡前习惯调整 | 我用它主要是改善睡眠环境，失眠问题建议咨询医生。 |
| 即时效果 | 立竿见影 | 对效果时间做确定承诺，缺少依据 | 短期感受是 / 长期仍需观察 | 短期感受是更方便收纳，长期耐用度还要继续观察。 |
| 站外交易 | 去淘宝搜同款更便宜 | 导向站外交易和价格对比可能触发引流/误导 | 平台内或官方门店可查看 / 以官方渠道为准 | 购买渠道建议看官方门店信息，我这里只记录体验。 |
| 医美暗示 | 院线级/医美级效果 | 普通护理或化妆品暗示医美效果，风险高 | 护理感更完整 / 使用步骤更接近专业流程的一部分 | 这套流程让我觉得护理步骤更完整，不等同于医美项目。 |
| 收益承诺 | 0成本月入过万 | 收益承诺和高收益诱导风险高 | 入门经验 / 收益受能力、时间和市场影响 | 这是我的副业入门复盘，收益受时间投入和市场情况影响。 |

### 2.9 推荐替代表达

**AI 改写规则：**
- `最/第一/唯一` → `在{限定场景}里更适合 / 我这次体验里表现更突出`
- `保证/一定/必` → `有助于 / 可能帮助 / 适合先尝试 / 结果因人而异`
- `治愈/根治/疗效` → `护理经验 / 健康管理 / 使用感受 / 建议咨询专业人士`
- `私信/加微信/外链` → `官方渠道 / 平台内入口 / 评论区交流问题`
- `点赞收藏` → `按需收藏 / 欢迎补充你的场景 / 你更在意 A 还是 B`

## 3. 微信内容方法论

### 3.1 平台特点

微信生态的核心不是单次曝光，而是信任、关系链和私域承接。公众号适合承载深度解释、品牌叙事、案例复盘和产品说明；朋友圈适合轻量触达；社群适合即时答疑和活动提醒；服务号更适合低频但高价值的服务通知，订阅号更适合持续内容心智。

平台规则上，微信公众平台运营规范明确关注诱导关注、诱导分享、违规导流、诱导点击、滥用模板消息、滥用客服消息、过度营销、不实信息、煽动夸大误导等风险。微信外链治理也强调防止违法违规、过度营销、诱导分享、危害用户利益等问题。

### 3.2 公众号文章结构

**公众号文章标准结构：**
```text
标题：问题 + 价值 + 场景
摘要：60 字内写清结论和适用人群
开头：真实场景 + 关键问题 + 本文给出的帮助
正文分层：问题 → 原因 → 方法 → 案例 → 边界
案例：背景 → 过程 → 结果 → 限制条件
CTA：一个主动作，使用官方或合规入口
结尾：总结 + 自然引导，不诱导分享/关注
```

**微信内容创作规则（至少 10 条）：**

1. 标题必须表达“读者为什么现在要看”，但不能用惊悚、命令式、隐藏关键信息的标题党。
2. 摘要/导语前 60 字写清楚文章结论、适用人群和阅读收益，避免空泛品牌口号。
3. 开头用一个真实业务场景，不直接上产品卖点；场景后必须提出一个可回答的问题。
4. 正文每个小标题写成结论句，例如“先判断预算，再选择服务包”，不要写“第一部分”“产品优势”。
5. 产品说明按「问题 → 原因 → 方案 → 证据 → 适用边界 → 下一步」顺序写。
6. 客户案例必须写明背景、目标、过程、结果和限制；不要写成普遍承诺，涉及客户信息要匿名或授权。
7. 公众号文章只设置一个主 CTA：预约咨询、查看活动、下载资料、加入社群四选一；不要多头跳转。
8. 服务号用于低频高价值通知和转化承接；订阅号用于持续内容心智；朋友圈用于轻触达；社群用于即时答疑。
9. 朋友圈文案控制在 80–150 字，先讲场景和价值，再给轻动作建议，不刷屏、不强迫互动。
10. 社群文案控制在 3 段：为什么发、对谁有用、怎么行动；连续营销信息之间至少插入答疑或价值内容。
11. 私域咨询话术只做信息收集和选项建议，不承诺结果；涉及医疗、金融、教育时主动加风险提示。
12. 复购唤醒写成维护提醒、使用技巧、会员权益提醒，不写“今天不买就亏”。

### 3.3 朋友圈/社群文案结构

**朋友圈结构：**
```text
第一句：场景痛点，不超过 25 字
第二句：给出判断标准或一个轻结论
第三句：说明资料/文章/活动适合谁
行动：看公众号/官方入口/预约，不强迫转发
```

**社群结构：**
```text
为什么发：今天补充一个{场景}常见问题。
对谁有用：适合{人群}，尤其是{条件}。
怎么行动：需要的可以看{官方入口/群文件/文章链接}，有问题直接在群里问。
```

**服务号/订阅号/朋友圈/社群差异：**
- 服务号：重服务、预约、通知、交易提醒。不要频繁营销。
- 订阅号：重内容、知识、品牌心智。适合系列化栏目。
- 朋友圈：重熟人信任。写短、写人话、写场景，不要像海报硬广。
- 社群：重即时互动。每条营销信息都要有信息价值或答疑价值。

### 3.4 私域转化写法

**私域转化链路：**
`内容建立认知 → 案例建立信任 → 清单降低决策成本 → 咨询确认匹配 → 服务承接 → 复购唤醒`

**AI 写法规则：**
- 咨询前：给清单，不催单。
- 咨询中：问需求、预算、时间、限制条件，不承诺结果。
- 咨询后：给 2–3 个选择和适用边界，不制造焦虑。
- 复购：用“使用提醒/维护建议/会员权益”代替“最后机会/不买就亏”。

### 3.5 行业案例

| 行业/场景 | 标题示例 | 开头示例 |
| --- | --- | --- |
| 餐饮 | 一家社区餐厅如何把“回头客”做成稳定增长 | 过去一个月，我们观察了 3 类回头客的消费路径：家庭聚餐、工作日简餐和周末社交。本文拆解可复用的触达方式。 |
| 本地生活 | 门店活动不要只发优惠，先解决这三个信任问题 | 很多活动不是没人看到，而是用户看到后仍不知道值不值得去。本文从信任、路径和售后说起。 |
| 美业 | 做项目前，顾客真正想确认的不是价格 | 美业咨询里，顾客反复问价格，背后其实是在确认安全、效果边界和服务流程。 |
| 教育 | 一节试听课，应该看哪些细节？ | 试听课不是一次成交表演，而是家长判断课程是否匹配孩子的窗口。本文提供一张可复用观察表。 |
| 家居 | 小户型改造前，先做一张生活动线表 | 比起直接买收纳柜，先记录三天生活动线，往往能更快找到空间浪费点。 |
| 服饰 | 为什么同一件衬衫，有人穿通勤有人穿休闲？ | 服饰内容转化的关键不是“好看”，而是帮用户判断版型、材质和场景。 |
| 旅游 | 三天两晚行程，如何避免越玩越累 | 行程规划不是景点堆叠，而是时间、交通、体力和天气的组合题。 |
| AI工具 | AI工具选型：先问工作流，再看功能列表 | 功能越多不一定越适合团队。选型前，先把要替代的人工流程写清楚。 |
| SaaS | 从一次客户案例，看SaaS内容如何建立信任 | SaaS 文章不要急着展示功能截图，先讲清客户的流程阻塞和决策约束。 |
| 私域 | 社群转化不是催单，而是减少用户决策成本 | 一个有效社群，应该让用户更快理解选择，而不是被频繁营销打扰。 |
| 活动 | 新品发布前，公众号应该先写哪三类内容？ | 新品上市不是一篇推文解决的事，预热、解释和答疑要分层完成。 |
| 复购 | 老客唤醒文案，为什么不要只发优惠券 | 老客复购往往需要的是提醒、理由和场景，而不是一次更大的折扣。 |

### 3.6 敏感词与风险表达

微信生态的主要风险不是单个词，而是“用户动作被强制或利诱”“关系链被过度营销打扰”“文章以标题党或软文方式误导用户”。因此，小加同学生成微信内容时应同时检查标题、正文、CTA、二维码/链接、社群话术和模板/客服消息。

| 风险类型 | 风险表达 | 风险原因 | 推荐替代表达 | 示例句子 |
| --- | --- | --- | --- | --- |
| 诱导分享 | 转发到朋友圈领资料 | 以利益或条件要求分享，属于诱导分享风险 | 文末提供公开资料入口 / 需要可自取 | 文末放了资料入口，需要的读者可以自行获取。 |
| 诱导关注 | 不关注不准看全文 | 强制关注后才能继续，损害用户选择 | 关注后可持续收到更新 / 不设阅读门槛 | 这篇完整内容可直接阅读，关注后会持续收到后续更新。 |
| 强承诺 | 保证成交 | 对商业结果做保证，容易构成夸大承诺 | 有助于提升转化效率 / 提供可验证方法 | 这套流程有助于减少咨询流失，实际转化因客群而异。 |
| 案例误导 | 100%成功案例 | 个案不能代表普遍效果，易误导读者 | 单个客户案例，不代表普遍效果 | 以下为一个客户的项目复盘，不代表所有项目都会获得同样结果。 |
| 金融收益 | 保本高收益 | 金融广告不得明示或暗示保本、无风险或保收益 | 投资有风险，收益不确定 / 仅作信息说明 | 本文仅做产品机制说明，投资有风险，收益不确定。 |
| 教育承诺 | 包过/不过退款 | 教育培训不得对考试、升学、证书结果作保证 | 提供备考方法和阶段测评，结果因人而异 | 课程提供阶段测评和答疑支持，考试结果与个人基础和投入相关。 |
| 权威背书 | 权威专家强烈推荐 | 无授权或无来源的专家背书风险高 | 引用公开资料并标注来源 / 访谈嘉宾观点 | 文中观点来自公开报告整理，链接已附在文末。 |
| 医疗体验 | 用户亲测三天治好 | 患者证言和治疗承诺风险高 | 用户反馈仅供参考，不能替代诊疗 | 以下反馈仅作体验参考，健康问题请以医生诊断为准。 |
| 绝对化用语 | 最佳解决方案 | 绝对化描述可能误导或贬损竞争者 | 一种可参考的解决路径 / 更适合某类场景 | 这是一种适合中小团队的流程改造路径。 |
| 夸张促销 | 史无前例优惠 | 夸张程度难证明，可能构成虚假宣传 | 本期活动权益 / 活动规则见说明 | 本期活动包含 2 项新增权益，规则见文末说明。 |
| 价格误导 | 原价999现价99 | 虚构原价、虚假折扣风险高 | 标注真实当前价、有效期和适用条件 | 当前体验价为 99 元，适用于 6 月 1 日前报名的新用户。 |
| 标题党 | 震惊/吓尿/必看 | 夸张惊悚词容易被判定为煽动、误导 | 这几个变化值得关注 / 一文看清 | 这 3 个变化会影响门店私域运营，建议先看清。 |
| 违规导流 | 文中反复放个人微信二维码 | 可能构成强导流、过度营销或私下交易风险 | 通过官方客服、小程序或合规表单咨询 | 需要咨询的读者可以通过文末官方客服入口预约。 |
| 诱导点击 | 头像红点/伪系统消息 | 仿冒系统提醒诱导点击，损害用户体验 | 使用正常按钮或文字链接 | 点击文末“阅读原文”查看活动规则。 |
| 模板消息滥用 | 向所有用户群发无关优惠 | 模板/客服消息不得发送与服务场景无关营销 | 只向订阅或咨询用户发送相关提醒 | 本提醒仅发送给已预约本服务的用户，用于确认时间。 |
| 多级分销 | 拉人头躺赚 | 多级分销、层级返利可能涉传销或平台违规 | 返佣规则合规透明，不承诺层级收益 | 如涉及推广奖励，请明确规则、资质和结算条件，不设计多级返利。 |
| 医疗健康 | 调理高血压/糖尿病 | 疾病治疗与疗效宣传属于高风险医疗表述 | 健康管理建议，具体诊疗请咨询医生 | 本文只讨论日常饮食记录方法，诊疗方案请咨询医生。 |
| 教育权威 | 命题老师押题必中 | 暗示考试机构人员参与或保证结果 | 老师经验分享和公开考纲解读 | 本课基于公开考纲做知识点梳理，不承诺命中考题。 |
| 市场地位 | 全城最低/第一家 | 市场地位和价格极值难以证明 | 本店/本期服务亮点 | 本店主打晚间可预约和透明报价，适合下班后到店。 |
| 焦虑营销 | 不看就亏/错过毁一年 | 用恐吓和焦虑推动购买，容易引发负面体验 | 适合某类人了解 / 可作为决策参考 | 如果你正在准备换系统，这份对比表可以作为决策参考。 |

### 3.7 推荐替代表达

**AI 改写规则：**
- `转发/关注后才能` → `公开阅读/公开领取/需要可自取`
- `保证成交/保证转化` → `有助于提升效率/减少试错/改善流程`
- `客户都成功了` → `单个案例，不代表普遍结果`
- `包过/必中/提分保证` → `阶段测评/方法训练/结果因基础和投入不同`
- `加微信下单` → `通过官方客服、表单、小程序或阅读原文了解`
- `不看就亏` → `适合正在处理{问题}的人参考`

## 4. 抖音内容方法论

### 4.1 平台特点

抖音内容以短视频兴趣推荐为主，核心是让用户愿意停留、看完、互动、转粉和搜索。公开报道提到抖音内容上传后会经过机器识别、人工审核与多轮评估，流量越高的内容可能经过更多评估；抖音社区自律公约也明确禁止虚假宣传、夸大效果或收益、隐瞒真实信息等宣传或引流行为。电商场景下，广告极限词、虚假宣传、功效夸大、价格误导、搬运侵权都是高风险点。

### 4.2 短视频脚本结构

**通用脚本：**
```text
0-3秒：可见动作/冲突画面 + 痛点钩子
3-8秒：指出误区或冲突
8-25秒：展示过程、证据、对比、步骤
25-40秒：给出结论和适用人群
40-60秒：自然提问或下一条预告
```

**抖音内容创作规则（至少 10 条）：**

1. 前 3 秒必须出现「可见动作/冲突画面 + 一句钩子」，不要用长铺垫和自我介绍。
2. 一条视频只讲一个结论；15–45 秒适合清单/教程/探店，45–90 秒适合测评/案例拆解，直播切片只保留一个冲突点。
3. 脚本按「钩子 → 痛点/冲突 → 过程/证据 → 结果/结论 → 自然提问」写，不能直接堆卖点。
4. 口播每句控制在 7–15 个汉字，字幕同步提炼关键词；每 4–6 秒给一个视觉变化。
5. 测评/教程必须拍出过程：开箱、菜单、后台、前后步骤、屏幕录制、到店动线等，减少纯口头结论。
6. 标题写搜索词，封面字写情绪/结论；标题可以长，封面字 6–10 字以内。
7. 评论区引导只能问真实问题，如“你更在意价格还是服务？”不写“评论 666 发资料”。
8. 本地/探店必须写清城市、商圈、时间、人均或价格区间、适合人群；视频里用字幕标注关键信息。
9. 直播话术每 3–5 分钟复述一次商品适用人群和限制条件；价格、赠品、库存以直播间公示为准。
10. 医疗、金融、教育、功效类内容必须降级为经验/知识/流程说明，并主动声明不构成诊疗、投资或考试承诺。
11. 禁止使用谐音、拆字、错别字、表情符号来规避审核；要从语义上改成真实、可证明、有限定条件的表达。
12. 使用素材前检查版权：画面原创或授权、音乐来自平台可用库、截图注明来源并避免大段搬运。

### 4.3 前 3 秒钩子写法

**钩子公式：**
1. `别急着{动作}，先看{关键点}`
2. `{人群}最容易忽略的不是{表面问题}，而是{真实问题}`
3. `我用{时间}测试了{对象}，结论是{有限定结论}`
4. `同样是{产品/服务}，差别在{维度}`
5. `如果你是{场景}，这{数字}点先确认`

**禁止钩子：** 震惊、爆了、全网第一、再不看就废了、评论 666、私信发你。

### 4.4 标题/封面/字幕写法

- **标题：** 写成用户会搜的句子，例如“AI 周报工具怎么选”“大理三天两晚路线”“梨形身材裤子推荐”。标题可稍长，但必须包含关键词。
- **封面字：** 6–10 字，突出结论，如“先看动线”“请客更稳”“AI周报流程”。
- **字幕：** 每条字幕只放一个意思；高风险词用语义降级，不用错别字规避。
- **口播：** 多用短句和动作词。每 4–6 秒切换画面或信息点。
- **评论区：** 用真实问题引导，如“你更在意价格还是服务？”不要用口令换资料。

### 4.5 评论区与转化引导

**合规评论引导模板：**
- “你更想看预算版还是详细版？”
- “你所在城市是哪里？我按城市补充清单。”
- “你更在意 A 还是 B？下一条按投票多的方向拆。”
- “如果你已经遇到这个问题，可以说下场景，我按场景给检查项。”

**不建议：**
- “评论 666 发教程”
- “点赞关注抽奖”
- “转发给 3 个好友领取”
- “私信我拿购买链接”

### 4.7 敏感词与风险表达

抖音的内容风险往往同时出现在口播、字幕、封面字、标题、评论区和直播话术中。小加同学生成视频脚本时，必须输出“口播稿 + 字幕稿 + 封面字 + 评论引导 + 合规备注”，并对每个字段做风险扫描。

| 风险类型 | 风险表达 | 风险原因 | 推荐替代表达 | 示例句子 |
| --- | --- | --- | --- | --- |
| 诱导互动 | 点赞关注马上抽奖 | 以利益诱导点赞关注评论，影响真实互动 | 想要清单可评论需求 / 我整理下一期 | 想要门店清单的，可以把城市留在评论区，我优先整理需求最多的。 |
| 诱导互动 | 评论666发教程 | 机械口令式诱导评论，互动质量低且有违规风险 | 评论你的使用场景，我按场景回复 | 你是新手还是团队运营？评论场景，我按场景补充。 |
| 减重承诺 | 3天瘦10斤 | 身体效果和时间承诺风险高 | 记录体重管理过程，结果因人而异 | 这是我的 14 天饮食记录，结果和基础代谢、运动量有关。 |
| 医疗功效 | 一招根治 | 疾病治疗保证和夸大效果风险高 | 一个可能有帮助的处理思路 | 这是一个日常护理思路，严重情况请及时咨询医生。 |

| 价格误导 | 全网最低下单 | 价格极值无法证明，直播/商品价格还会变化 | 活动信息以商品页或直播间公示为准 | 当前活动以商品页显示为准，我只讲适合人群和使用感受。 |

| 强迫传播 | 不转不是中国人 | 道德绑架式转发引导风险高 | 欢迎分享给需要的人 | 如果你身边有人正在装修，可以分享给他参考。 |
| 站外导流 | 私信拿网址 | 导向私下交易或站外链接有风险 | 通过主页或官方入口了解 | 产品入口在主页官方链接，视频里先讲适用场景。 |

| 贷款承诺 | 贷款无门槛秒放款 | 金融贷款广告不得承诺放款、忽略资质和利率条件 | 申请条件和利率以机构公示为准 | 申请条件、年化利率和审批结果以持牌机构公示为准。 |
| SaaS承诺 | 永久免费无任何限制 | 服务条款变化和限制条件未披露，易误导 | 提供免费版或试用期，具体以服务条款为准 | 这款工具有免费版，功能范围以官网服务条款为准。 |
| 用户评价 | 0差评 | 评价绝对化且可能无法证明 | 收到的用户反馈里这些点被提到较多 | 用户反馈里被提到较多的是上手快和模板多。 |
| 资质夸大 | 国家认证第一 | 国家级、第一等极限词和资质夸大风险 | 展示真实资质名称、编号和适用范围 | 如有资质，只写证书名称、编号和适用范围，不做排名夸大。 |

### 4.8 推荐替代表达

**AI 改写规则：**
- `强刺激表达` → `具体痛点 + 可见动作`，如“再不学就废了”改成“不会这个步骤会影响效率”。
- `功效承诺` → `过程记录 + 个体差异`，如“秒瘦”改成“记录体重管理过程，结果因人而异”。
- `收益承诺` → `方法拆解 + 风险提示`。
- `诱导互动` → `真实问题`。
- `站外导流` → `主页官方入口/商品页公示/平台内说明`。
- `低俗擦边` → `功能、版型、材质、过程、场景表达`。

## 5. 三平台通用风险词库

| 风险类型 | 典型风险表达 | AI 改写规则 |
| --- | --- | --- |
| 绝对化用语 | 最、第一、顶级、唯一、永久、100% | 改为限定条件 + 可验证依据，如“在我这次试用中”“适合某类场景” |
| 功效承诺类 | 必瘦、必白、立竿见影、永久有效 | 改为体验表达 + 个体差异，如“我个人感受”“长期效果需观察” |
| 医疗健康类 | 治愈、根治、药效、医生推荐、无副作用 | 改为健康管理/护理经验；涉及诊疗时建议咨询专业医生 |
| 金融收益类 | 稳赚、保本、无风险、躺赚、收益翻倍 | 改为风险提示 + 信息说明，不给投资建议和收益承诺 |
| 教育升学类 | 包过、押题必中、提分保证、名校必进 | 改为方法、陪跑、阶段测评，不承诺考试或升学结果 |
| 价格促销类 | 最低价、最后一天、原价虚标、限量虚假 | 写清活动时间、条件、库存和价格来源 |
| 引流诱导类 | 私信拿链接、加微信、扫码领、转发领取 | 使用平台内官方入口、阅读原文、客服或合规表单 |
| 低俗擦边类 | 性暗示、身体羞辱、夸张露骨标题 | 转为产品功能、版型、场景和审美表达 |
| 侵权搬运类 | 搬运影视、盗图、洗稿、未授权音乐 | 使用原创/授权素材，注明来源，避免大段复制 |
| 虚假权威类 | 国家认证、专家背书、官方内部消息 | 展示真实证书、公开来源和适用范围，不编造身份 |

**通用审核前置流程：**
```text
1. 判断平台：小红书/微信/抖音。
2. 判断行业：是否涉及医疗、医美、药品、保健食品、金融、教育、未成年人。
3. 扫描 9 类风险：绝对化、功效、医疗、金融、教育、价格、诱导、低俗、侵权。
4. 对每个风险表达给出：风险原因 + 推荐替代表达 + 改写后示例。
5. 若缺少证据，删除数字和承诺；若涉及高敏行业，要求资质和合规确认。
6. 输出内容时保留适用边界和免责声明。
```

## 7. 可直接复用的创作模板

### 7.0 Markdown 交付约定

生成小红书笔记 Markdown 时，`note.md` 只能包含可直接发布的笔记内容：标题、封面字、正文、结尾、话题标签。不要把创作建议、合规检查、风险提示、修改说明、素材缺口或运营建议混进 `note.md`。

如需输出建议或合规内容，单独创建 `note_review.md`、`compliance_notes.md` 或同类独立文档。该文档可以包含：合规检查表、风险表达替换、内容优化建议、素材缺口、发布注意事项。

### 7.1 小红书笔记模板

```text
平台：小红书
类型：种草/避坑/测评/清单/教程/体验/对比
输入：行业、目标人群、场景、痛点、产品/服务、证据素材、限制条件
输出：
1. 标题：{人群/场景} + {痛点} + {有限定结果}
2. 封面字：{6-10字结论}
3. 开头：我在{场景}遇到{痛点}，这次按{维度}试了一遍。
4. 正文：3-5 个要点，每点包含观察、证据、建议。
5. 边界：适合谁/不适合谁。
6. 结尾：自然提问，不诱导点赞收藏。
7. 标签：品类词 + 场景词 + 人群词 + 地点/行业词。
合规检查：另写到 note_review.md，不放进 note.md。
```

### 7.2 微信公众号文章模板

```text
标题：{问题}：{解决方向}，先看这{数字}点
摘要：本文适合{人群}，解决{问题}，提供{清单/案例/方法}。
开头：真实场景 + 关键问题 + 本文价值。
正文：
一、问题是什么
二、为什么会发生
三、解决方案
四、案例复盘：背景/过程/结果/限制
五、适用边界
CTA：通过官方入口{预约/下载/咨询}
结尾：总结，不诱导分享关注。
```

### 7.3 朋友圈文案模板

```text
{场景痛点}，很多人第一反应是{常见误区}。
```

4. **模板生成模块**
   按平台输出结构化模板，而不是自由作文。每次生成必须包含：
   - 内容标题
   - 开头/钩子
   - 正文/脚本
   - CTA
   - 标签或评论引导
   - 合规备注
   - 可替换字段

## B. 小加同学的默认生成流程

```text

→ 选择内容类型
```
